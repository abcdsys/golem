# GG 插件

基于 gogpu/gg 库的文字转图片插件，支持将文本内容转换为精美的图片，并完美支持 Emoji 表情渲染。

## 功能特性

- **文字转图片**：将任意文本内容转换为 PNG 图片
- **Emoji 支持**：完美渲染彩色 Emoji 表情符号
- **中文字体**：使用 MapleMono Nerd Font 字体，支持中文和特殊符号
- **自定义颜色**：支持自定义字体颜色和背景颜色
- **自动换行**：智能处理长文本，自动换行排版
- **透明背景**：支持透明背景，方便二次编辑

## 使用方法

插件通过 `text.to.image` 能力被调用，将文本转换为图片。

### 基础用法

```
文本内容
```

插件会自动将文本转换为图片，使用默认样式（黑色字体，透明背景）。

### 自定义颜色

通过消息参数可以自定义颜色：

```json
{
  "capability": "text.to.image",
  "args": {
    "context": "你好，世界！👋",
    "font_color": "#FF0000",
    "bg_color": "#FFFFFF"
  }
}
```

## 参数说明

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `context` | string | 是 | - | 要转换的文本内容 |
| `font_color` | string | 否 | black | 字体颜色 |
| `bg_color` | string | 否 | transparent | 背景颜色 |

### 颜色格式

支持多种颜色格式：

#### 1. 预定义颜色名

```
black   - 黑色
white   - 白色
red     - 红色
green   - 绿色
blue    - 蓝色
```

**示例**：
```json
{
  "font_color": "white",
  "bg_color": "black"
}
```

#### 2. 十六进制格式

支持以下格式：
- `#RGB` - 短格式（3 位），如 `#F00` 表示红色
- `#RGBA` - 短格式（4 位），如 `#F00F` 表示不透明红色
- `#RRGGBB` - 标准格式（6 位），如 `#FF0000` 表示红色
- `#RRGGBBAA` - 完整格式（8 位），如 `#FF0000FF` 表示不透明红色
- `0xRRGGBBAA` - 十六进制前缀格式，如 `0xFF0000FF`

**示例**：
```json
{
  "font_color": "#FFFFFF",      // 白色
  "bg_color": "#1E1E1EFF"       // 深灰色
}
```

**透明度说明**：
- Alpha 通道范围：`00`（完全透明）~ `FF`（完全不透明）
- 默认值：不指定时为 `FF`（不透明）
- 透明背景：设置为空字符串或使用 `#RRGGBB00`

## 样式配置

### 默认样式参数

插件内置以下样式参数（不可修改）：

- **字体大小**：24px
- **最大宽度**：1600px
- **行间距**：0.8
- **内边距**：25px
- **主字体**：MapleMono-NF-CN-Regular.ttf
- **Emoji 字体**：NotoColorEmoji.ttf

### 文本处理

- **自动换行**：超过最大宽度时自动换行
- **保留换行符**：文本中的 `\n` 会被保留为换行
- **Emoji 渲染**：自动识别并使用彩色字体渲染 Emoji
- **混合排版**：文字和 Emoji 可以混合使用

## 技术细节

### 字体特性

#### MapleMono Nerd Font
- 等宽字体，适合代码和对齐文本
- 包含 Nerd Font 图标集
- 完整的中文支持
- 清晰的渲染效果

#### Noto Color Emoji
- Google 官方 Emoji 字体
- 支持最新的 Emoji 标准
- 彩色位图渲染
- 自动缩放适配字体大小

### 渲染机制

1. **文本分段**：自动识别 Emoji 和普通文字
2. **独立渲染**：Emoji 使用专用字体渲染
3. **智能混排**：保持字符对齐和视觉统一
4. **高质量输出**：使用 Catmull-Rom 插值算法缩放 Emoji

### 性能优化

- 字体文件嵌入二进制，无需外部文件
- 高效的文本测量和换行算法
- 最小化内存分配
- PNG 编码输出

## 输出规格

- **格式**：PNG
- **颜色模式**：RGBA
- **透明度**：支持
- **尺寸**：根据文本内容自动计算
  - 宽度 = 内边距 × 2 + 最长行宽度
  - 高度 = 内边距 × 2 + 行数 × 行高

## 限制说明

1. **字体固定**：不支持自定义字体
2. **样式固定**：字体大小、行间距等参数固定
3. **宽度限制**：最大宽度 1600px，超出自动换行
4. **纯文本**：不支持富文本格式（粗体、斜体等）
5. **内嵌字体**：字体文件较大（约 30MB），首次加载较慢

## 常见问题

### 1. Emoji 显示不正常

**原因**：
- Emoji 字体损坏
- 不支持的 Emoji 版本

**解决方法**：
- 确保使用标准 Emoji 字符
- 避免使用最新的 Emoji（可能字体未更新）

### 2. 中文乱码

**原因**：
- 字体文件缺失
- 编码问题

**解决方法**：
- 确保 MapleMono 字体文件完整
- 使用 UTF-8 编码

### 3. 图片尺寸过大

**原因**：
- 文本内容过长
- 换行过多

**建议**：
- 适当精简文本内容
- 使用换行符分段
- 考虑分多张图片

### 4. 颜色不符合预期

**检查清单**：
- 确认颜色格式正确
- 检查 Alpha 通道值
- 使用标准的十六进制格式

### 5. 字符对齐问题

**说明**：
- Emoji 和文字高度可能不完全一致
- 这是混合字体排版的正常现象
- 插件已尽量优化对齐效果
